# State 2 — Rules and Invariants

## Purpose

Separate always-true constraints and lifecycle policy from configurable values
after the domain models are stable.

## Status

**Not started**

## Expected outputs

- Data, ownership, access, and integration invariants.
- Exact owning responsibility for every invariant.
- Valid lifecycle transitions where the product defines them.
- Rules, policies, fallback decisions, and runtime configuration candidates.
- Traceable invariant-ledger entries with stable identifiers.

No rules or invariant landings are proposed in this bootstrap.
